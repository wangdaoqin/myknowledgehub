# 知海导航 - 部署和使用说明

## 网站结构

知海导航是一个静态网站，包含以下页面：

- `index.html` - 主页，包含导航和资源预览
- `category/` - 分类页面
  - `programming.html` - 编程语言分类
  - `statistics.html` - 统计分析分类
  - `ai.html` - AI 工具教程分类
  - `accounting.html` - 会计知识教程分类
- `about.html` - 关于本站页面
- `admin.html` - 管理后台页面
- `dashboard.html` - 学习看板页面
- `updates.html` - RSS更新页面
- `friends.html` - 好友专区页面

## 部署方法

### 方法 1: GitHub Pages

1. **创建 GitHub 仓库**
   - 在 GitHub 上创建一个新的仓库
   - 将本地代码推送到仓库

2. **启用 GitHub Pages**
   - 进入仓库设置 → Pages
   - 选择 `main` 分支作为源
   - 选择根目录 `/` 作为发布目录
   - 点击 "Save" 保存设置

3. **访问网站**
   - 部署完成后，GitHub Pages 会提供一个网址，格式为 `https://username.github.io/repository-name`

### 方法 2: Vercel

1. **导入项目**
   - 访问 [Vercel](https://vercel.com/)
   - 登录或注册账号
   - 点击 "Add New Project"
   - 选择从 GitHub 导入仓库

2. **配置部署**
   - 选择仓库后，Vercel 会自动检测静态网站配置
   - 保持默认设置，点击 "Deploy"

3. **访问网站**
   - 部署完成后，Vercel 会提供一个域名，格式为 `project-name.vercel.app`

### 方法 3: Netlify

1. **导入项目**
   - 访问 [Netlify](https://www.netlify.com/)
   - 登录或注册账号
   - 点击 "Add new site" → "Import an existing project"
   - 选择从 GitHub 导入仓库

2. **配置部署**
   - 选择仓库后，保持默认设置
   - 点击 "Deploy site"

3. **访问网站**
   - 部署完成后，Netlify 会提供一个域名，格式为 `site-name.netlify.app`

## 如何更新网站

### 更新内容

1. **修改静态文件**
   - 直接编辑对应的 HTML 文件
   - 保存修改后，重新部署到托管平台

### 添加新资源

1. **更新分类页面**
   - 打开对应的分类 HTML 文件（如 `category/programming.html`）
   - 在 `resources-grid` 部分添加新的资源卡片

2. **更新主页**
   - 打开 `index.html`
   - 在 "今日推荐" 或 "最近更新" 部分添加新的资源

## 使用指南

### 浏览资源

1. **通过分类浏览**
   - 在主页点击分类卡片或导航链接
   - 浏览对应分类下的所有资源

2. **搜索资源**
   - 在主页或分类页面的搜索框中输入关键词
   - （注：静态网站的搜索功能需要手动实现，目前仅为占位符）

### 管理资源

1. **访问管理后台**
   - 点击导航栏中的 "管理后台" 链接
   - 在管理后台页面管理资源（注：静态网站的管理功能需要后端支持，目前仅为前端界面）

### 学习追踪

1. **使用学习看板**
   - 点击导航栏中的 "学习看板" 链接
   - 查看学习进度和计划（注：静态网站的学习追踪功能需要后端支持，目前仅为前端界面）

### 查看更新

1. **查看 RSS 更新**
   - 点击导航栏中的 "RSS更新" 链接
   - 查看最新的行业资讯和学习资源（注：静态网站的 RSS 功能需要后端支持，目前仅为前端界面）

### 好友功能

1. **访问好友专区**
   - 点击导航栏中的 "好友专区" 链接
   - 与好友共享学习资源和经验（注：静态网站的好友功能需要后端支持，目前仅为前端界面）

## 扩展建议

1. **添加后端功能**
   - 使用 Next.js API 路由实现动态功能
   - 连接数据库存储资源和用户数据

2. **增强用户体验**
   - 添加响应式设计，适配移动设备
   - 实现真正的搜索功能
   - 添加资源收藏和评论功能

3. **优化性能**
   - 压缩静态文件
   - 使用 CDN 加速资源加载
   - 实现懒加载和缓存策略

## 技术栈

- **前端**：HTML5, CSS3, JavaScript
- **样式**：自定义 CSS（深色主题）
- **字体**：系统字体（PingFang SC, Microsoft YaHei）
- **托管**：支持 GitHub Pages, Vercel, Netlify 等静态网站托管服务
